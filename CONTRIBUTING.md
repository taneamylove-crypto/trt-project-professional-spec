# Contributing

TRT uses an RFC-first workflow.

## Rules

- Do not implement business logic before the relevant RFC is approved.
- Keep protocol, operator, foundation, and institute concerns separated.
- Add tests for every approved implementation change.
- Treat legal, security, and randomness concerns as launch blockers, not polish.

## Commit style

Use concise conventional commits:

- `chore: ...`
- `docs: ...`
- `ci: ...`
- `feat: ...` only after RFC approval
- `fix: ...`
